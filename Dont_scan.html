When using Symantec Data Center Security, you can exclude files, folders, and extensions from active or full scans to improve performance and avoid unnecessary scanning of safe items. Here are some suggestions:

System Files: Exclude critical system files like pagefile.sys and hiberfil.sys.

Temporary Files: Exclude directories such as C:\Windows\Temp and C:\Users\\AppData\Local\Temp.

Backup Files: Exclude folders containing backup files that are already scanned, such as C:\Backup.

Database Files: Exclude database files like .mdf and .ldf for SQL Server, or .db files for other databases.

Virtual Machine Files: Exclude virtual machine files like .vmdk and .vmem.

Log Files: Exclude log files that are frequently updated, such as C:\Logs.

Software Development Files: Exclude directories containing source code or build artifacts, such as C:\Projects.
